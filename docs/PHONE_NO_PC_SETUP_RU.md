# Work Boost OS без ПК: запуск для телефона

Цель: пользоваться приложением с телефона без запущенного сервера на домашнем ПК.

## 1. Что уже подготовлено в проекте

В проект добавлено:

1. PWA-режим (`manifest.webmanifest`, `sw.js`, иконки).
2. Мобильный экран `Commander (моб.)`.
3. Переключатель `Моб. режим / Полный`.
4. Настройка `Backend URL` в боковой панели.
5. Docker-образ для облачного запуска (`Dockerfile`).
6. Серверный набор зависимостей (`backend/requirements-server.txt`).
7. Поддержка `WORKBOOST_DATA_DIR` в backend для хранения данных вне кода.

## 2. Сценарий «без ПК» (рекомендуемый)

Нужно один раз развернуть backend+frontend в облаке.

Дальше телефон работает напрямую с облаком.

## 3. Вариант A: Docker на любом VPS

### 3.1. Сборка образа

```bash
docker build -t work-boost-os .
```

### 3.2. Запуск контейнера

```bash
docker run -d \
  --name work-boost-os \
  -p 8000:8000 \
  -e GOOGLE_API_KEY=YOUR_KEY \
  -e WORKBOOST_DATA_DIR=/data \
  -v workboost_data:/data \
  work-boost-os
```

Что это даёт:

1. Приложение доступно по `http://<server-ip>:8000`.
2. История, ops-store и бэкапы сохраняются в volume `workboost_data`.

## 4. Вариант B: Railway/Render/Fly (managed)

Общий порядок:

1. Подключить репозиторий с проектом.
2. Выбрать `Dockerfile` в корне `ai_studio_clone`.
3. Добавить env:
   1. `GOOGLE_API_KEY`
   2. `WORKBOOST_DATA_DIR=/data`
4. Добавить persistent disk/volume и примонтировать в `/data`.
5. Получить публичный URL сервиса.

После этого:

1. Открой URL на телефоне.
2. Добавь на главный экран (`Install app`).

## 5. Как подключить телефон

### 5.1. Если фронт и backend на одном домене

Ничего не настраивать не нужно:

1. Открыл URL.
2. Поставил на главный экран.
3. Готово.

### 5.2. Если фронт отдельно, backend отдельно

1. Открой приложение.
2. Перейди в `Чат` -> Sidebar.
3. В блоке `Backend URL` укажи адрес backend (например `https://api.your-domain.com`).
4. Нажми `Применить`.

## 6. UX-компромисс для телефона

Чтобы не утонуть в перегруженном UI:

1. На телефоне используй `Commander` как основной режим.
2. Полный Ops Hub открывай точечно, когда нужна детализация.
3. Глубокую настройку скоринга и большие разборы делай на ноутбуке/планшете.

Рабочая схема:

1. Телефон: intake, быстрые решения, weekly check-in, writer.
2. Десктоп: аналитика, playbook-тюнинг, массовые правки данных.

## 7. Как использовать эффективно на телефоне

1. Утро (2-3 минуты): открыть Commander, проверить KPI и риски.
2. Днём на ходу: закинуть новый лид в быстрый intake.
3. Перед откликом: открыть Writer и собрать черновик cover letter.
4. Вечером: короткий weekly check-in и backup.

## 8. Ограничения и риски

1. Без облака «полностью без ПК» невозможно, т.к. backend и Gemini-вызовы должны где-то выполняться.
2. На бесплатных облачных тарифах может быть sleep/cold start.
3. Для PWA-install в некоторых браузерах нужен HTTPS.
4. Не храни продовый API key в открытом репозитории.

## 9. Минимальный чек-лист запуска

1. `GOOGLE_API_KEY` задан.
2. Сервис отвечает по `/api/config`.
3. `/api/ops/phase1` отдает JSON без 500.
4. На телефоне открывается UI и работает `Writer`.
5. Проверен backup и запись в `.ai/ops/phase1_store.json` (через volume `/data`).
